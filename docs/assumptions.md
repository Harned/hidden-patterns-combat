# Assumptions (MVP)

1. Для кодирования признаков используется токен-поиск по названиям колонок (русские/английские ключевые слова).
2. Многорядные заголовки Excel читаются как MultiIndex и "сплющиваются" в строковый вид `lvl1 | lvl2 | ...`.
3. HMM в MVP использует `GaussianHMM` (hmmlearn) на векторе компактных кодов + временных/результативных признаков.
4. Число скрытых состояний по умолчанию равно 3, но задается параметром CLI.
5. Интерпретация состояния в MVP строится по средним значениям признаков внутри каждого скрытого состояния.
6. Скрытые состояния (`latent states`) в коде задаются отдельно от инженерных признаков; признаки — это наблюдения, а не сами состояния.
7. Для работы с реальными последовательностями используется grouping в `lengths` (contiguous sequence ids), что является практическим MVP-решением для hmmlearn.
8. В текущем MVP `sequence_id` формируется приоритетно по `sheet + athlete_name`; при недоступности полей используется безопасный fallback (`athlete`, `sheet`, `episode`, `sequence_0`).
9. При редких конфигурациях, где `hmmlearn` дает нулевые суммы строк в `transmat_`, применяется нормализующий repair шаг для сохранения работоспособности модели.
